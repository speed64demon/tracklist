Spotify API Credentials Spotify

You need a free Spotify Developer app to get a Client ID. This takes about 3 minutes.

1
Open Spotify Developer Dashboard

Log in with your Spotify account.

↗ developer.spotify.com/dashboard
2
Create a new App

Click Create App. Name it anything (e.g. "Playlist Bridge"). For the Redirect URI, enter exactly: http://localhost:3000/callback — you will paste a different URI below in a moment.

3
Set the Redirect URI to this page

In your Spotify app settings, add this page's URL as a Redirect URI. The easiest option: use this page's file:// URL (shown after you open this file). You can also use http://localhost:8888/callback if you serve it locally.

4
Copy your Client ID

From the app's Settings page, copy the Client ID and paste it below.

The Spotify Client ID is not secret — it just identifies your app. You don't need a Client Secret for the PKCE auth flow used here.

Tip: If opening as a local file, use http://localhost:8888/callback and run a tiny local server, OR deploy this file to any static host and use that URL.

YouTube Data API Credentials YouTube

You need a Google Cloud project with the YouTube Data API v3 enabled. Free quota is 10,000 units/day — enough for hundreds of songs.

1
Open Google Cloud Console ↗ console.cloud.google.com
2
Create a new project

Click the project dropdown → New Project → name it anything.

3
Enable YouTube Data API v3

APIs & Services → Library → search "YouTube Data API v3" → Enable.

4
Create OAuth 2.0 credentials

APIs & Services → Credentials → Create Credentials → OAuth client ID → Web application. Add the same URL as your Authorized Redirect URI (matching what you used for Spotify, or this page's URL). Copy the Client ID.

5
Configure OAuth consent screen

Under OAuth consent screen, set to External, add your own email as a test user. Add the scope https://www.googleapis.com/auth/youtube.

The app stays in "Testing" mode which is fine — you're the only user. No need to publish it.
